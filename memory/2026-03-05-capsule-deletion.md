## 问题分析

### 预期功能
清理 markdown 输入并验证飞书文档块结构，解决：
- 文档写入时的结构错位
- 前后段落穿插
- 400 错误

### 实际测试
经过 **8 轮测试** 发现：
1. markdown 格式 → 块顺序完全打乱
2. 纯文本格式 → 块顺序仍然打乱
3. write 动作 → 顺序打乱
4. append 动作 → 顺序打乱

### 根本原因
**飞书文档 API 的内容解析器会随机打乱块的顺序**，这是 API 本身的行为，不是 Capsule 能解决的问题。

### 测试证据
输入顺序：
```
标题 → 第一段 → 第二节 → 第二节内容 → 列表1/2/3 → 子节 → 子节内容
```

实际输出顺序（随机示例）：
```
第二节内容 → 子节 → 第一段 → 第二节 → 文档结束 → 子节内容 → 标题 → 列表
```
