# 工具使用错误分析与改进方案

## 错误记录

### 错误1：重复使用Brave Search而非Tavily
**发生时间**：2026-03-11 10:29
**错误描述**：用户昨天已明确要求使用Tavily作为默认搜索工具，今天又复现了同样的错误
**根因分析**：
1. 每次会话重启后，工具偏好没有被自动加载
2. 系统默认的web_search工具是Brave Search
3. 没有机制在会话开始时自动检查用户的工具偏好

### 错误2：忘记Tavily API密钥
**发生时间**：2026-03-11 10:29
**错误描述**：昨天已配置好的Tavily API密钥，今天再次忘记
**根因分析**：
1. API密钥存储在环境变量中，会话重启后丢失
2. 没有将密钥持久化到配置文件
3. 没有启动检查清单来验证必要配置

---

## 改进方案

### 方案A：创建Session Startup Checklist（推荐）
创建一个启动检查文件，每次会话开始时自动执行：

```
1. 读取 MEMORY.md 获取工具偏好
2. 读取 ai-memory-system/Intent/preferences/tools.md 获取工具配置
3. 验证必要的环境变量/配置是否就绪
4. 设置会话级别的工具别名
```

### 方案B：创建Tools Preference Skill
创建一个专门的skill来管理工具偏好：
- 存储默认搜索工具配置
- 存储API密钥引用（不存储明文，存储获取方式）
- 提供工具选择辅助函数

### 方案C：使用Reflect Skill自动化
启用reflect的auto-reflect功能，自动提取和持久化工具偏好
