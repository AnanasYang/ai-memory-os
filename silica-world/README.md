# 🌊 Silica World - 硅基虚拟世界

一个物理上可信的多智能体虚拟世界，用于观察涌现行为。

## 🎯 核心特性

### 物理公平性
- 所有角色共享相同的硅基物理定律
- **速度 × 时间 = 距离** - 刚性约束，无"行动点"抽象
- 碰撞由动量守恒自然处理，无需外部仲裁

### 准实时制
- 世界以固定tick推进（默认2 tick/秒）
- 每个tick = 1分钟虚拟时间
- 物理定律自动处理所有交互

### 5层记忆架构
```
L0 状态层 → 当前位置、能量、视野
L1 情境层 → 最近遭遇的事件
L2 行为层 → 习惯策略（伏击/逃跑）
L3 认知层 → 对环境规律的理解
L4 核心层 → 价值观（生存、力量/敏捷）
```

### 用户交互
- 可随时接入控制任意角色
- 自然语言命令（向左、加速、休息等）
- 无输入时角色由AI自主运行

## 🚀 快速开始

### 启动Web界面
```bash
cd silica-world
python3 run.py
```

访问 http://localhost:8080

### 命令行模式
```bash
python3 run.py --cli
```

## 🎮 使用指南

### Web界面
1. 点击"🐟 初始化鱼缸"创建角色
2. 点击"▶ 启动世界"开始运行
3. 点击角色卡片选择要控制的角色
4. 在命令框输入自然语言指令

### 支持的命令
- `向左游` / `向右游` / `向上游` / `向下游`
- `加速` / `减速`
- `休息`
- 任意自然语言（会记录但可能不执行）

- **世界地图**: 实时显示角色位置和移动方向
- **角色状态**: 能量、速度、位置、记忆数量

- **控制面板**: 向角色发送命令

## 🧠 设计理念

### 涌现而非预设
我们不设计"大鱼吃小鱼"的剧情，而是设定：
- 大鱼质量大、速度慢、视野广
- 小鱼质量小、速度快、视野窄
- 碰撞时大鱼有动量优势，小鱼有速度优势

**观察**: 在这种物理约束下，会涌现什么样的生存策略？

### 记忆的演化
角色不是状态机，而是会积累记忆：
- 被追过3次 → L2形成"逃跑习惯"
- 成功伏击2次 → L2形成"伏击偏好"
- 这些习惯会改变未来的决策

### 物理即公平
没有时间特权，没有行动点货币：
- 思考需要时间（等待玩家输入时世界继续运行）
- 移动消耗能量（能量需要休息恢复）
- 所有约束来自物理定律本身

## ⚙️ 技术参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 世界尺寸 | 100m × 100m | 2D连续空间 |
| Tick率 | 2/s | 控制算力消耗 |
| 虚拟时间/tick | 1分钟 | 1秒现实 = 2分钟虚拟 |
| 大鱼速度 | 4 m/min | 慢但强壮 |
| 小鱼速度 | 6 m/min | 快但脆弱 |
| 视野范围 | 30m | 感知限制 |
| 记忆上限 | 50条 | L1情境记忆 |

## 📝 哲学笔记

> "我们不创造故事，我们创造故事的土壤。"

这个系统的目标是观察**简单规则产生的复杂行为**，而非设计预设的剧情。

**核心问题**:
- 物理约束下的最优生存策略是什么？
- 记忆会如何改变行为模式？
- 长期运行会出现什么样的生态平衡？
