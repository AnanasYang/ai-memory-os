# 日报/周报生成逻辑说明

## 分层信息架构

### 核心层（可直接落地）
**定义**：可被企业规模化开发使用的开源项目、模型、新范式

- 新的训练方法、推理优化
- 数据开发工具
- 可落地的 AI 工具

### 近层（行业前沿，值得跟踪）
**定义**：行业动态、产品发布、技术路线选择

- 主机厂技术路线选择
- 供应商方案
- 产品发布、融资、合作

### 外延层（拓展视野）
**定义**：前沿探索、观点碰撞

- 上游技术突破
- 学术观点与思辨

## 分类规则

系统根据文章标题和分类自动归类：

| 关键词 | 分类 |
|--------|------|
| GPT/模型/AI/智能体/Agent | 大模型/AI技术 |
| 自动驾驶/智驾/端到端/蔚来/小鹏/特斯拉 | 自动驾驶 |
| 机器人/具身/Figure | 机器人 |
| 发布/上线/推出 | 产品发布 |
| 收购/融资/盈利 | 企业动态 |

## 报告模板

### 日报结构
1. 今日概览（统计信息）
2. 核心层（详细报道）
3. 近层（标题列表）
4. 分析与建议（针对传统合资车企）

### 周报结构
1. 本周概览
2. 本周十大要闻
3. 分层汇总
4. 深度分析（技术趋势 + 针对性建议）
5. 下周关注重点

## 定制化建议

针对 Bruce 的背景（传统合资车企、资源有限、组织活力不足）：

1. **强调可落地性**：优先报道开源工具、可试用方案
2. **关注头部节奏**：重点跟踪特斯拉、小鹏、华为等
3. **成本意识**：避免推荐需要大量资源投入的方案
4. **渐进推进**：建议从小规模试点开始
