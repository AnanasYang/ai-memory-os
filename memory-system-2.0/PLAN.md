# Memory System 2.0 重构计划

## 项目目标
重构五层记忆系统 + Dreams机制 + Memory OS，建立流畅的记忆流转管道

## 核心问题清单

### P0 - 阻塞性问题（必须修复）
1. L0→L1 粒度过粗（天级→事件级）
2. L1→L2 晋升逻辑模糊（频率→模式归纳）
3. L2→L3 跃迁路径缺失
4. L4 死层（无沉淀机制）
5. Weekly Dream 未运行
6. Memory OS 数据层与文件系统脱节

### P1 - 体验问题
7. Review 流程形式化
8. 缺乏冲突检测机制
9. Dreams 非叙事性
10. 缺乏元认知层

---

## 阶段规划

### Phase 1: 核心机制重构（优先级：P0）
**目标**：修复记忆流转的基础逻辑

**任务 1.1**: L0→L1 会话级拆分
- 设计话题切换检测算法
- 实现事件级 L1 生成器
- 替换现有 Daily Dream 的 L1 生成逻辑

**任务 1.2**: L1→L2 模式归纳引擎
- 设计"相似情境→相似反应"检测
- 实现 L2 候选自动生成
- 创建简化版 Review 确认流程

**任务 1.3**: L2→L3 跃迁逻辑
- 定义"多习惯→认知框架"转换规则
- 实现 L3 候选检测
- 建立认知网络初始结构

**任务 1.4**: L4 激活与校准
- 设计 L4 填充工作坊流程
- 实现 L3→L4 冲突检测
- 建立价值观校准机制

---

### Phase 2: Dreams 机制重构
**目标**：建立有效的 Review 管道

**任务 2.1**: Daily Dream 2.0
- 改为事件级摘要格式
- 增加叙事性结构
- 关联目标/项目标签

**任务 2.2**: Weekly Dream 激活
- 修复 cron 任务
- 实现 L2 候选汇总
- 创建一键确认界面

**任务 2.3**: Monthly/Quarterly Review
- 自动化 L2→L3 审查
- 生成认知网络演进报告
- L3→L4 冲突报告

---

### Phase 3: Memory OS 重新设计
**目标**：构建与文件系统同步的可视化界面

**任务 3.1**: 数据层重构
- 设计 Markdown↔JSON 同步管道
- 实现实时文件监听
- 建立统一数据模型

**任务 3.2**: API 层设计
- /api/memories (CRUD)
- /api/dreams (生成/查询)
- /api/reviews (确认/晋升)
- /api/insights (模式检测)

**任务 3.3**: 核心组件重写
- Memory Galaxy 2.0 (真实数据)
- Intent Orbit (关联目标)
- Evolution Timeline (层级演进)
- Insights Radar (冲突检测)

**任务 3.4**: Review 界面
- Weekly Review Dashboard
- L2 候选确认界面
- L3 框架演进可视化

---

### Phase 4: 整合与部署
**目标**：端到端测试与上线

**任务 4.1**: 数据迁移
- 迁移现有 L1-L2 数据
- 生成初始 L3 框架
- 引导填充 L4

**任务 4.2**: 集成测试
- 全链路流转测试
- Review 流程验证
- Memory OS 数据同步测试

**任务 4.3**: 部署与监控
- 部署 Memory OS
- 配置 Cron 任务
- 建立健康监控
